# Аудит проекта Nuvio (Codex)

## 1. Архитектурные расхождения
Проект заявлен как Rust Workspace (разбиение на `domain`, `ai`, `bot`, `migrations`), но текущая реализация — монолитный бинарник в `backend/`. 
- **Рекомендация**: Переход к структуре workspace позволит изолировать логику бота и AI, что упростит тестирование и масштабирование.

## 2. Критические риски исполнения
- **Конфигурация Docker**: Текущая настройка может конфликтовать с путями модулей в монолите.
- **Интеграция Telegram**: В `main.rs` отсутствует полноценная диспетчеризация обновлений; сейчас обработчик webhook-а — это просто заглушка, возвращающая "ok".

## 3. Риски Telegram / Gemini
- **Логика функций**: `add_task` не имеет обработки ошибок (error handling) при вызове Gemini API, что может привести к падению бэкенда при сбоях сети или API.
- **Безопасность**: Middleware валидации `initData` требует проверки подписи HMAC для каждого запроса; текущая реализация нуждается в аудите на предмет возможности "replay" атак.

## 4. Приоритетный Roadmap
1. Рефакторинг структуры проекта в сторону Rust Workspace (согласно плану).
2. Настройка `teloxide` диспетчера для реальной обработки сообщений вместо заглушки.
3. Добавление `Result` обработки в Gemini-инструменты для предотвращения падений.
4. Вынос конфигурации в более безопасное хранилище (вместо `.env` для продакшена).
